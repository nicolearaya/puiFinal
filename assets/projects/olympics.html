The Olympics have long been hailed as the highest standard of athletic achievement. During a Harvard computer science course on data visualization, I worked in a team to build a website that challenges the idea that the Games are a simple meritocracy by contextualizing medal wins with social, economic, and political factors.
1/ PROBLEM DEFINITION

We first defined our target audience to be the general public, assuming minimal background knowledge of Olympics history. We wanted to invite users to compare themselves to the athletes through interactive elements of our story and encourage them to develop insights from: 1) cross-country comparisons of records/wins, 2) an analysis of how social, economic, and political factors influence the games, and 3) individual-level data that portray what it takes to become an Olympic athlete.

To do so, we researched available datasets regarding Olympics history, individual-level athlete data, census economic data, and global economic data. From these resources, we sketched visualizations to build our data story and used Tableau to further explore trends and correlations.

2/ DATA AND STORYBOARD

We initially included gender inequalities, geographic distribution, economic data, and political events among the relevant factors, but found it challenging to format the audience takeaways into a cohesive data story. Given the scope of the project and the limitations of available data, we decided to narrow our focus to the effect of economic inequalities on Olympic results. Our data was more granular for the United States than for other countries, so we needed to communicate these localized findings without extrapolating globally.

We created our data storyboard with the following narrative structure: hook, rising insights, main message, and resolution. Our hook allowed users to compare their own biological data to that of Olympic athletes, creating a personalized connection to the data story. The story then transitions from individual-level US athlete data to country-level inequalities, culminating in sport accessibility rankings based on the diversity of winners and athletes.

3/ PROTOTYPE

The next step was to prototype the website based on our data storyboard. I built a website framework using HTML and CSS, and created a Github repository to facilitate team collaboration. We utilized object-oriented Javascript programming and D3 interactive visualization libraries to create a class for each visualization type. Within each class we maintained a consistent, readable code structure of initialization, data wrangling, and visualization updating. Our final prototype included a choropleth map, scrollable data table, line and area charts, bubble chart, and parallel coordinate chart.

4/ TEST AND REVISE

We then tested our data story design with user interviews and think aloud studies. We received feedback on the intuitiveness of the user interface, clarity of the data messages, and effectiveness of the visual encodings. We found that users sometimes did not find the information encoded in the tooltips, so we made clear indications when to hover or moved the dynamic data to the page. The feedback also indicated that our data sources were not always clear, so we applied a header system to each visualization to consistently signal the timespan and location of the data. Using visual design guidelines including whitespace, C.R.A.P (contrast, repetition, alignment, and proximity), and Gestalt principles, we refined the interaction designs to emphasize each visualization's data message and relevance to Olympics accessibility.
